# Jane's Learning World

An adaptive learning tutor for one little kid. Voice-driven, theme-rotating, generates fresh activities forever, never repeats.

![JaneOS demo](video/JaneOS_demo.mp4)

## What it does

- 7 themed worlds (unicorns, mermaids, dinos, space, cats, horses, Bluey)
- 60–90s activity bursts: math, sight words, phonics, reading fluency, place value, science, social, SEL, letter tracing
- Adaptive: every attempt updates a per-skill mastery score; weak skills get more practice
- Streak-based reward: 3 right in a row triggers a confetti celebration
- Visible feedback flash on every answer so audio isn't required
- Parent dashboard at `/parent` shows mastery, accuracy, and recent sessions

## Stack

- **Backend:** Python 3.12, aiohttp, on port 9100
- **LLM:** Claude (via the Claude Code CLI for Max-plan billing, no per-token API charges)
- **TTS:** [Piper](https://github.com/rhasspy/piper) running locally with the `en_US-libritts_r-medium` neural voice; server returns WAV bytes, browser plays them so it works on remote devices too
- **Frontend:** vanilla HTML/CSS/JS, no build step
- **Storage:** SQLite at `data/jane.db`
- **Always-on:** managed by `launchd`, autostarts on boot, KeepAlive=true

## Run

```bash
./setup.sh         # creates .venv, installs aiohttp + piper-tts
mkdir -p voices && cd voices
curl -LO https://huggingface.co/rhasspy/piper-voices/resolve/v1.0.0/en/en_US/libritts_r/medium/en_US-libritts_r-medium.onnx
curl -LO https://huggingface.co/rhasspy/piper-voices/resolve/v1.0.0/en/en_US/libritts_r/medium/en_US-libritts_r-medium.onnx.json
cd ..
./launch.sh
# Open http://localhost:9100
```

For autostart on a Mac, copy `com.janeos.server.plist` into `~/Library/LaunchAgents/` and `launchctl load` it.

## Architecture

- **Activity bank** (`activity_bank.py`): 158 hand-curated activities across 11 skills. Always available, instant responses, works offline.
- **LLM queue** (`server.py`): a background worker pre-generates fresh Claude-authored activities into a small queue so `/api/next` returns in <100 ms. Falls back to the bank if the LLM is slow or unavailable.
- **TTS worker** (`server.py`): single-consumer `asyncio.Queue` ensures speech segments never overlap. Piper synthesis runs in a thread executor.
- **Schema validation**: every LLM response is checked against the click-only schema before serving to avoid mid-flight surprises (no math problems with theme emojis prefixed, no count tasks without visible emojis, etc.).
